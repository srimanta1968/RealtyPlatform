1 · Phase Goal

Phase 1 captured leads. Phase 2 makes the operational machine real: pipeline stages enforced by workflow, visits scheduled and executed in-system, tasks generated automatically (rule-based), basic analytics so leadership answers "how are we doing?" without asking anyone.

In Scope

Out of Scope
Deferred

5 · Visit Lifecycle Workflow (Temporal)

flowchart LR
  S[scheduled] --> CONF[send confirmation]
  CONF --> R24[reminder T-24h]
  R24 --> R2[reminder T-2h]
  R2 --> CHECK{outcome}
  CHECK -->|completed| CAP[trigger outcome capture task]
  CHECK -->|no_show| RES[create reschedule task]
  CHECK -->|cancelled| REENG[create re-engagement task]
  CAP --> END[end]
  RES --> END
  REENG --> END

Lead Nurture Cadence (rule-based NBA)

on stage = QUALIFIED + no activity for 48h → create "follow-up call" task
on stage = VISITED + no follow-up in 24h → escalate to lead owner manager
on stage = NEGOTIATION + no movement for 5d → leadership digest

Rules live in a versioned config inside CRM Service. Phase 6 replaces with an LLM agent.

9 · Deliverables

  • Visit Service deployed
  • CRM Service upgraded with task engine + SLA policies
  • Analytics Service + Day-1 dashboards live
  • WhatsApp Business API integration certified (templates approved)
  • Temporal cluster + event bus operational
  • Runbook updated; SLO targets published per service
  • Staff training: visit scheduling, task inbox, outcome capture

10 · Exit Criteria

  • ≥95% of scheduled visits have a recorded outcome within 24h
  • ≥80% of qualified leads have an open task during business hours
  • Leadership funnel dashboard adopted as the weekly review artifact
  • Median follow-up time from enquiry: <4h
  • SLA breach rate <10% in steady state
