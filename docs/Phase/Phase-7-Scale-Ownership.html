Scale & Ownership

From a managed service to a Kiana-owned, hybrid-operated platform — and the blueprint for the next vertical. The bridge that turns this product into a Vertical AI OS.
Duration: 8–12 weeks (overlapping steady state)
Adds: Hardening · Governance · Vertical OS extraction
Outcome: Sustainable hybrid ownership
P1 · Trust P2 · Ops P3 · Growth P4 · Field P5 · Premium P6 · AI P7 · Scale

1 · Phase Goal

Phases 1–6 built the platform. Phase 7 is about operational maturity and strategic optionality:

  • Transition from "Insignia-managed" to "Kiana-operated, Insignia-supported" (hybrid)
  • Institutionalize training, governance, and continuous improvement
  • Harden the platform for multi-tenancy / multi-region scale
  • Extract the Vertical AI OS core so it can be replicated (Insurance, Healthcare next)

2 · Vertical OS Extraction

Refactor the codebase into three layers. Tenants (customers) plug their config into a vertical pack; vertical packs plug into the platform core.

Layer 3 · Tenant Config
Kiana / Next Customer
branding · taxonomies · integrations · SLA policies · prompt overrides
Layer 2 · Vertical Pack
Real Estate Pack
Lead / Property / Visit / Campaign domain models · agents · prompts · dashboards
(future packs: Insurance · Healthcare)
Layer 1 · Platform Core
Domain-agnostic
event bus · workflow engine · AI gateway · agent runtime · notification · media · document · user/role
A new vertical (e.g., Insurance) reuses Layer 1 and adds its own Layer 2.

9 · Customer Success Function

Net-new role(s) at Kiana (with Insignia coaching):

  • Activation specialist for portal adoption
  • Account engagement reviews for top customers
  • Win/loss analysis loop into product

10 · Replication Playbook

The deliverable that makes the next vertical possible. Documents:

  • How to stand up a new vertical pack on Platform Core
  • Reference data models, prompts, dashboards (templated)
  • Onboarding checklist for a new tenant
  • Cost model + sizing table
  • Validated end-to-end (paper exercise) by an Insignia squad outside the Kiana team

12 · Deliverables

  • Transition plan executed; ownership matrix signed off
  • Training program delivered; competency checks passed
  • Governance forums operating with documented agendas and decisions log
  • Platform hardening: load tests passed at 5× volume; DR drill completed
  • SLA & support framework live; on-call rotations established
  • Customer Success function staffed and producing weekly reports
  • Vertical OS extraction complete; code separation merged; documentation published
  • Replication playbook published

13 · Exit Criteria

  • Kiana operates day-to-day for 30 consecutive days without Sev-2+ Insignia escalation
  • All governance forums have run for two full cycles with documented outcomes
  • Replication playbook validated end-to-end by an outside Insignia squad
  • KPI trend lines positive: lead capture, conversion, cost-per-qualified-lead, sales cycle
  • Customer NPS for portal ≥40; field NPS for app ≥50
  • Annual penetration test passed; findings remediated or risk-accepted

15 · Strategic Outcome

By the end of Phase 7, Kiana Realty Growth Platform is no longer a "project" — it is:

  1. A productized real estate operating system owned by Kiana, supported by Insignia
  2. A reference implementation of the Insignia Vertical AI OS pattern
  3. A repeatable engine that turns intent into revenue, with the playbook to replicate into the next vertical

Real estate OS → Insurance OS → Healthcare OS

Phase 7 is the bridge that makes that thesis credible to the next customer.
